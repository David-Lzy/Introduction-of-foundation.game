# 翻译进度追踪策略（多语言）

本文用于约定如何在本项目中追踪不同语言版本（当前为中文 / 英文，未来可扩展）的翻译进度，避免目录混乱、状态不一致、重复翻译。

## 一、目标

追踪对象包括：

* 各章节是否已翻译
* 翻译完成度（未开始 / 进行中 / 已完成 / 待校对 / 已发布）
* 与中文原文是否同步（原文更新后是否需要重译）
* 负责人 / 更新时间 / 备注

## 二、追踪原则

### 1）中文为源语言（Source of Truth）

* `Chinese/` 作为内容基准
* 所有其他语言版本都以对应中文文件为追踪单位
* 如果中文文件结构变化，以中文为准更新翻译状态

### 2）按“文件粒度”追踪，不按段落追踪（默认）

默认以单个 Markdown 文件为最小追踪单位，原因：

* 成本低，维护简单
* 适合当前项目章节型结构
* 后续如果某个文件特别长，再单独拆分段落级追踪

### 3）路径镜像

不同语言目录尽量保持相同结构，便于自动比对与人工查找。

例如：

* `Chinese/发育/4.主线与开荒.md`
* `English/Growth/4.Mainline-and-Early-Expansion.md`（目录名与文件名均可翻译，但建议保留编号）

建议规则：

* 保留编号（如 `0.` `1.` `2.`）
* 目录层级一致
* 目录名建议翻译（如 `发育` -> `Growth`），避免混用中英文目录
* 如文件名翻译后差异较大，必须在进度表中记录“源文件路径”

## 三、状态定义（统一口径）

建议使用以下状态值，避免自由发挥：

* `not_started`：未开始翻译
* `draft`：已开始翻译，内容未完成
* `translated`：已完成翻译（未校对）
* `reviewing`：校对中（术语、格式、链接检查）
* `published`：已完成并可对外使用
* `outdated`：中文原文已更新，当前译文可能过期
* `skipped`：暂不翻译（低优先级或废弃内容）

## 四、建议维护文件

在 `Translate/` 下维护两类文件：

### 1）策略文件（你现在正在看）

* `Translate/翻译进度追踪策略.md`
* 用于定义规则，不记录具体进度

### 2）进度总表（核心）

建议新建：

* `Translate/translation-status.md`

用于记录每个源文件在不同语言下的状态。

## 五、进度表结构（推荐）

建议用 Markdown 表格，字段如下：

* `模块`：如 发育 / 资源 / 活动 / 商会 / 氪金 / 打架
* `中文源文件`：`Chinese/...`
* `语言`：`English` / `Japanese` / `Korean` ...
* `目标文件`：`English/...`
* `状态`：使用上文定义的状态值
* `版本同步`：`synced` / `source_updated`
* `最后更新`：日期（`YYYY-MM-DD`）
* `负责人`：可选
* `备注`：术语争议、待确认点、跳过原因

## 六、推荐工作流（实际执行）

### 1）新增中文文档时

1. 在 `Translate/translation-status.md` 新增该条目
2. 为每种目标语言创建状态（通常先 `not_started`）
3. 标记优先级（可写在备注中）

### 2）开始翻译时

1. 创建目标语言文件（或占位文件）
2. 状态改为 `draft`
3. 填写负责人与开始日期（可写在备注）

### 3）翻译完成时

1. 状态改为 `translated`
2. 检查标题层级、列表、内部链接、术语一致性
3. 校对时改为 `reviewing`
4. 校对完成后改为 `published`

### 4）中文原文更新时（非常关键）

1. 在中文文件修改后，检查受影响语言条目
2. 将相关语言状态改为 `outdated`
3. 在备注中写明“更新原因”（例如：新增小节、数值改动、措辞修正）
4. 修订译文后恢复为 `translated` / `published`

## 七、版本同步策略（避免“假完成”）

建议在进度表中增加“中文最后修改日期”或“同步标记”，至少做到以下之一：

方案 A（简单，手动）：

* `版本同步` 字段使用 `synced` / `source_updated`

方案 B（更稳，推荐）：

* 增加 `中文更新时间` 字段
* 增加 `译文同步到` 字段
* 两者不一致时自动判定为 `outdated`

## 八、命名与规范建议

### 1）文件名

* 中文保留原命名
* 英文建议保留编号前缀，便于排序
* 尽量避免只靠中文和英文文件名“猜对应关系”

### 2）术语表（后续建议）

当语言版本增多时，建议新增：

* `Translate/glossary.md`

维护常用术语统一翻译，例如：

* 主线任务
* 反攻行动
* 活跃里程碑
* 商运
* 公会/商会（统一用词）

## 九、最低落地方案（现在就能用）

如果你希望先快速跑起来，先做这三件事即可：

1. 保留本文作为规则说明
2. 新建 `Translate/translation-status.md`
3. 先只追踪 `Chinese/` -> `English/` 的文件级状态

这样后续扩展日文、韩文时只需要加列或加条目，不需要重做规则。
