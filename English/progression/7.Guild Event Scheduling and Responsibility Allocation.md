# Guild Event Scheduling and Responsibility Allocation

> Audience: MIT2 / HuaXia guild leadership  
> Focus: locking in event timing, ownership, escalation boundaries, and conflict priorities  
> Updated: 2026-05-02

## One-line conclusion

**MIT2 currently works best when controllable recurring events are organized around UTC 12:00, while high-intensity PvP is handled separately according to live event windows and wartime command.**

---

## 1. Core principle

Guild Leader DavidLi should retain final authority over:

- final combat command
- final diplomatic positioning
- major conflict decisions
- key leadership role changes

Event owners should mainly handle:

- routine execution
- reminders
- opening or scheduling events
- organizing main and substitute rosters
- post-event review and logging

In short:

> **Execution can be delegated. Major PvP and diplomacy cannot become uncontrolled.**

---

## 2. Why UTC 12:00 should be the core anchor

The strongest stable management window is currently **UTC 09:00-14:00**.

Why:

- 35釐米 is consistently available
- Aurora·極光 is consistently available
- GoldenFnx often becomes reachable after about UTC 11:00
- SuzieQ may also be available for admin and English communication during that span

That makes the lowest-friction operating rhythm:

- management coverage: UTC 09:00-14:00
- recurring officer-opened events: as close to UTC 12:00 as possible
- high-intensity PvP: follow live system windows instead of forcing a fixed slot

---

## 3. Recommended operating cadence

### 3.1 Fixed management window

Treat **UTC 09:00-14:00** as the main duty window for:

- member questions
- event reminders
- guild mail follow-up
- cooldown checks
- roster arrangement
- pre-battle notices

### 3.2 Fixed recurring event block

The most valuable block to standardize first is:

- **UTC 12:00: Arms Race**
- **around UTC 12:40-13:00: Defend the Trade Route**

Why this pairing works:

- both are officer-opened events
- both benefit from concentrated attendance
- members are already online and fleets are ready
- it creates a stable memory pattern for the guild

### 3.3 High-intensity PvP should stay separate

These events should not be hard-bound to UTC 12:00:

- Shadowfront
- VS Day 6 / Duel Battlefield

For those events, the rule should be:

- follow the live event panel first
- prefer windows with stronger officer coverage
- do not force a main roster into a weak or uncertain time slot

## 5. Activity priority order

When multiple activities collide, use this priority:

1. **VS Day 6 / Duel Battlefield**
2. **Shadowfront**
3. **Arms Race**
4. **Defend the Trade Route**
5. **Daily maintenance / social activity**
6. **Non-urgent diplomacy**

Core decision rules:

- low-intensity events must not steal command attention from high-intensity PvP
- officer-opened events can be delayed; high-risk PvP cannot be improvised badly
- any action with diplomatic risk should be confirmed before execution
